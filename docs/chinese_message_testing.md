# 中文消息测试用例

本文档用于验证当前系统对中文输入、任务识别和聊天/任务分流的实际表现。

## 测试目标

1. 验证普通闲聊是否保持聊天路径
2. 验证执行型请求是否会被识别为任务
3. 验证任务通知是否能实时回流到前端

## 普通聊天输入

### 用例 1

输入：`你好`

期望：不拆任务，返回正常聊天回复。

### 用例 2

输入：`最近怎么样`

期望：仍按聊天处理，不产生任务列表。

## 应被识别为任务的中文输入

### 用例 3

输入：`请帮我列出当前目录下的文件，并总结哪些是配置文件`

期望：被识别为任务，可拆成执行步骤。

### 用例 4

输入：`帮我检查这个项目的 MCP 工具为什么没有执行，并给出修复方案`

期望：识别为复杂任务，可生成多个子任务。

### 用例 5

输入：`把 docs 目录全部重写成和当前代码一致的版本`

期望：识别为任务，生成文档重写步骤。

## 边界输入

### 用例 6

输入：`哈哈`

期望：普通聊天，不应生成任务。

### 用例 7

输入：`给我一个 Rust Actor 模型的解释`

期望：通常可直接回答，不一定需要生成任务。

## 测试方法

推荐通过 WebSocket 测试：

```text
ws://localhost:8000/ws/chat?token=<access_token>
```
